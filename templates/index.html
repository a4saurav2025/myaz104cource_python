Azure basics & tools

Administration tooling

  • Portal — GUI for ad‑hoc management, dashboards, notifications; not ideal for repetitive tasks.
  • Azure CLI — Cross‑platform automation. Example: az vm create --resource-group RG --name VM1 --image UbuntuLTS
  • PowerShell (Az) — Windows‑centric scripting. Example: New-AzVM -ResourceGroupName RG -Name VM1 -Image UbuntuLTS

Azure Resource Manager (ARM)

  • Central management layer for create/update/delete with consistent RBAC, locks, tags.
  • Scopes: Management Group → Subscription → Resource Group → Resource.
  • Templates enable declarative, repeatable deployments with dependency ordering.

Cost optimization

  • Use Pricing Calculator & Cost Management; save via Reservations, Spot VMs, Hybrid Benefit, and Dev/Test pricing.

Security posture & advice

  • Defender for Cloud (formerly Security Center): recommendations, secure score, threat protection.
  • Azure Advisor: best‑practice guidance across cost, reliability, performance, ops excellence, security.

Identity, RBAC & Policy

Microsoft Entra ID (Azure AD)

  • Cloud identity using OAuth/OIDC/REST; hybrid via Azure AD Connect (hash sync / PTA / federation).
  • Security: Conditional Access, Identity Protection, MFA, named locations.

RBAC essentials

  • Role assignment = principal + role definition + scope.
  • Built‑in roles (Owner/Contributor/Reader + data‑plane roles); deny assignments override allows.

Azure Policy

  • Define/assign policies and initiatives; evaluate at create/update and every 24h.
  • Common: Allowed locations, not‑allowed resource types, regulatory blueprints.

Service Health

  • Personalized incidents, health history, RCAs, and alerting for your regions/services.

Storage services

Blob Storage

  • Blob types: Block (streaming), Append (logs), Page (VHD).
  • Access tiers: Hot/Cool (account), Archive (blob‑level; requires rehydration).
  • Redundancy: LRS, ZRS, GRS/RA‑GRS, GZRS/RA‑GZRS; lifecycle management & soft delete.

Files, Disks, Queues, Tables

  • Azure Files: SMB/NFS shares; use Premium for IO‑intensive; snapshot for backup.
  • Managed Disks: Standard HDD/SSD, Premium SSD, Ultra (data disk only, huge IOPS).
  • Queues: simple decoupled messaging (64 KB msg, 500 GB queue).
  • Tables: NoSQL key/attribute store; partition+row key design.

Key Vault

  • Secrets/Keys/Certs with RBAC or access policies; versioning and audit; data vs management plane.

  • Backup: Azure Backup, ASR, or snapshots.

App Service

  • PaaS for web/APIs; plans: Free/Shared/Basic/Premium/Isolated; slots, scaling, custom domain/SSL.

Containers

  • ACR (Basic/Standard/Premium; roles AcrPush/AcrPull; Content Trust; geo‑replication).
  • ACI for simple isolated containers; VNet, probes, container groups.
  • AKS for orchestration: node pools, deployments, services/ingress, PVs.

Virtual networking & delivery

VNet fundamentals

  • Address spaces & subnets; NICs; NSGs (priority: lower = earlier); peering (no overlapping ranges).
  • Azure DNS (public/private); 168.63.129.16 for DHCP/DNS wire server.

Security & routing

  • Azure Firewall: stateful, rule collections, Threat Intel, centralized policy (hub‑spoke).
  • DDoS Protection: Basic (free) vs Standard (credits, metrics, DDRT, multi‑layer).

Load balancing & gateways

  • Load Balancer (L4), App Gateway (L7 + WAF), Front Door (global HTTP), Traffic Manager (DNS‑based).
  • Connectivity: VPN Gateway (S2S/P2S, IPsec) and ExpressRoute (private circuit, BGP).

CDN

  • Edge caching; Microsoft/Akamai/Verizon SKUs; define origin, TTL, HTTPS, rules.

Monitoring & threat protection

Azure Monitor

  • Metrics + Logs (KQL) → Alerts, Autoscale; Workbooks, Insights (App/VM/Storage/AKS).

Microsoft Sentinel

  • SIEM/SOAR on Log Analytics: connectors, analytics rules, workbooks, hunting, automation.

SQL Defender & AIP

  • Defender for SQL: vulnerability assessment & ATP alerts (SQLi, anomalous access).
  • Azure Information Protection: label, classify, protect, and track documents.
